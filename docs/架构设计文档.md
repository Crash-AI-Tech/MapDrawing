# Map 架构设计文档

> **版本**: v6.0 — OpenNext 迁移版
> **日期**: 2025-07-26
> **核心目标**: 高性能地图绘画、多笔刷支持、面积墨水系统、圖釘留言、全量历史保留
> **技术基座**: TypeScript + Next.js 16 (App Router) + MapLibre GL + **Cloudflare (D1 + R2 + OpenNext)**

---

## 一、系统概述

Map 是一个「在真实地图上画画」的全球协作平台。用户可以：

- 🎨 在 MapLibre GL 地图上使用四种画笔（铅笔/马克笔/喷枪/荧光笔）进行涂鸦
- 📌 在地图上放置彩色图钉，留下 50 字以内的短消息
- 🌍 所有创作实时同步，路过其他城市时能看到来自世界各地的涂鸦和留言
- 🖨️ 面积墨水系统公平约束每个用户的绘画量

### 设计原则

1. **单一平台** — 数据库、存储、计算、CDN 全部在 Cloudflare，内网访问，零跨境延迟
2. **引擎与框架解耦** — `src/core/` 为纯 TypeScript，零 React 依赖，通过 hooks 桥接
3. **边缘优先** — Next.js 运行于 Cloudflare Workers (via OpenNext + Assets)，全球 300+ PoP 就近响应
4. **自建认证** — Lucia Auth v3 + D1，完全掌控用户数据，无第三方依赖
5. **渲染管线分离** — MapLibre GL（地图）与自定义 Canvas overlay（绘画）完全独立
6. **公平墨水** — 面积 × 缩放等级的墨水消耗公式，鼓励精细绘画，惩罚大面积覆盖

### 数据流

```

  PointerEvent → InputManager → DrawingEngine.moveStroke()
    ├─ InkManager.calculateSegmentCost() → 扣减墨水
    ├─ Brush.onStrokeMove() → 绘制到 activeCanvas
    └─ endStroke() → StrokeManager.add() + HistoryManager.push()
         ├─ SyncManager.broadcastStroke()

  Server Action → D1 查询 → Lucia createSession → Set-Cookie
```

## 五、Drawing Engine 核心设计

### 5.1 引擎与框架完全解耦

`src/core/` 目录下的所有代码为**纯 TypeScript**，零 React / Next.js 依赖。通过以下桥接层连接：

```
   React 世界                         纯 TS 世界
┌──────────────────┐              ┌──────────────────┐
│ useDrawingEngine │──initWithMap─│  DrawingEngine   │
│ (hooks)          │              │  StrokeManager   │
│                  │              │  InkManager      │
│ Zustand stores   │◀──events───│  HistoryManager  │
│ (drawingStore,   │              │  ViewportManager │
│  inkStore, etc.) │              │  RenderPipeline  │
│                  │              │  InputManager    │
│                  │──subscribe─▶│  BrushRegistry   │
└──────────────────┘              └──────────────────┘
```

`useDrawingEngine` hook 在 MapLibre 地图加载完成后：

1. 创建 `DrawingEngine` 实例
2. 创建 `MapLibreAdapter` 桥接坐标转换
3. 创建 `WebCanvasProvider` 叠加两层 Canvas 到地图上
4. 创建 `RenderPipeline` 并启动 rAF 渲染循环
5. 创建 `InputManager` 监听 Pointer Events
6. 订阅引擎事件 → 更新 Zustand stores

### 5.2 笔画数据模型

```typescript
interface StrokePoint {
  x: number;           // 经度 (存储时) / 屏幕 X (绘制时)
  y: number;           // 纬度 (存储时) / 屏幕 Y (绘制时)
  pressure: number;    // [0, 1]
  timestamp: number;   // ms
}

interface StrokeData {
  id: string;                // UUID v7 (时间有序)
  userId: string;
  userName: string;
  brushId: string;           // pencil | marker | spray | highlighter
  color: string;             // #RRGGBB
  opacity: number;           // [0, 1]
  size: number;              // 基准尺寸 (CSS px)
  points: StrokePoint[];     // 存储时为 geo 坐标 (lng, lat)
  bounds: GeoBounds;         // 空间包围盒
  createdZoom: number;       // 创建时的缩放级别
  createdAt: number;         // Unix ms
}
```

### 5.3 笔画生命周期

```
startStroke(screenX, screenY, pressure)
  → canDraw() 检查墨水 → 创建 UUID v7 → Brush.onStrokeStart()

moveStroke(screenX, screenY, pressure)  (每个 PointerMove)
  → 计算 pixelDistance → calculateSegmentCost() → 累加 inkAccumulator
  → 整数部分扣减墨水 → Brush.onStrokeMove()
  → 墨水耗尽自动 endStroke()

endStroke()
  → 扣减剩余分数墨水 → 计算 GeoBounds → 屏幕坐标转换为经纬度
  → StrokeManager.add() → HistoryManager.push() → emit stroke:end
```

### 5.4 笔刷架构

四种笔刷均继承 `BaseBrush` 抽象类，通过 `BrushRegistry` 管理：

| 笔刷 | ID | 渲染技术 | 特殊效果 |
|------|-------|---------|---------|
| 铅笔 | `pencil` | `quadraticCurveTo` 贝塞尔曲线 | 压力感应变宽 |
| 马克笔 | `marker` | OffscreenCanvas 双 pass 技术 | 同笔画不叠色 |
| 喷枪 | `spray` | 半径内随机粒子散点 | 高斯衰减密度，确定性回放 |
| 荧光笔 | `highlighter` | `globalCompositeOperation='multiply'` | 固定 opacity=0.4 |

扩展笔刷只需 `extends BaseBrush` + `registry.register()`。

### 5.5 渲染管线（三层 Canvas）

```
MapLibre GL (WebGL 矢量地图)
         │
         │  叠加
         ▼
┌─────────────────────────────────┐
│  compositeCanvas                │  ← RenderPipeline.render() 每帧合成
│  ┌──────────────────────────┐   │
│  │  历史笔画 (直接渲染)      │   │  ← StrokeRenderer 遍历可见笔画
│  └──────────────────────────┘   │
│  + activeCanvas (当前笔画)       │  ← drawImage() 叠加实时绘制层
└─────────────────────────────────┘
```

**渲染逻辑（每帧 rAF）：**

1. 清空 compositeCanvas
2. 从 StrokeManager 查询视口内笔画 (R-tree 空间查询)
3. 过滤：`currentZoom >= stroke.createdZoom - 3` (缩放隐藏规则)
4. 按 `createdAt` 升序排列（新笔画在上层）
5. 对每个笔画调用 `Brush.renderFullStroke()`，传入 geo→screen 变换
6. 以 `storedSize * 2^(currentZoom - createdZoom)` 计算屏幕像素大小
7. 叠加 activeCanvas（当前正在绘制的笔画）

### 5.6 缩放与笔画显示规则

| 规则 | 实现 |
|------|------|
| 最低绘画缩放 | zoom >= 18 才能画 (`MIN_DRAW_ZOOM`) |
| 最低图钉缩放 | zoom >= 20 才能放 (`MIN_PIN_ZOOM`) |
| 笔画隐藏 | zoom < stroke.createdZoom - 3 时隐藏 |
| 笔画缩放 | `screenSize = storedSize * 2^(currentZoom - createdZoom)` |

## 六、墨水系统设计

### 6.1 概述

墨水系统是 Map 的核心公平机制。每个用户拥有 100 点墨水，绘画消耗墨水，墨水随时间恢复。

| 参数 | 值 | 说明 |
|------|------|------|
| 最大墨水 | 100 | MAX_INK |
| 恢复速度 | +1 / 18秒 | REGEN_INTERVAL_MS = 18000 |
| 平衡常数 K | 20 | INK_COST_K |
| 基准缩放 | 18 | ZOOM_BASE |
| 图钉消耗 | 50 | PIN_INK_COST |

### 6.2 墨水消耗公式

```
segmentCost = brushSize * pixelDistance * 2^(2 * (18 - zoom)) / K
```

- `brushSize` — 当前画笔大小 (1~10)
- `pixelDistance` — 当前线段的屏幕像素长度
- `zoom` — 当前地图缩放级别
- `K = 20` — 平衡常数（调大 = 更便宜，调小 = 更贵）

**关键设计：zoom 每降低一级，消耗 x4（二次方缩放）**

这意味着：
- 在 zoom 18 画一笔 100px (size=3) → 消耗 `3*100/20 = 15` 墨水
- 在 zoom 19 画同样一笔 → 消耗 `3*100*0.25/20 = 3.75` 墨水
- 在 zoom 20 画同样一笔 → 消耗 `3*100*0.0625/20 = 0.9375` 墨水

**效果：鼓励精细绘画（高 zoom + 小笔），惩罚大面积覆盖（低 zoom + 大笔）。**

### 6.3 浮点墨水累加器

DrawingEngine 中使用 `inkAccumulator` 浮点数累加分数墨水消耗：

```
每个 moveStroke() → segmentCost 累加到 inkAccumulator
当 inkAccumulator >= 1 → 扣减 floor(accumulator) 整数部分
endStroke() → 扣减剩余分数部分（不四舍五入，保持精确公平）
```

### 6.4 墨水持久化与离线恢复

墨水状态存储在 `localStorage`，包含当前值和时间戳。页面重新加载时：

```
恢复墨水 = savedInk + floor(elapsed / REGEN_INTERVAL) * REGEN_AMOUNT
```

即使离线或关闭浏览器，墨水也会按时间计算恢复。

## 七、图钉系统设计

### 7.1 概述

用户可以在地图上放置彩色图钉，附带一条 50 字以内的短消息。

### 7.2 交互流程

1. 用户点击工具栏的图钉按钮 → 进入放置模式
2. 地图鼠标指针变为对应颜色的图钉 SVG 光标
3. 点击地图某处 → 弹出 `PinPlacer` 面板（输入消息 + 选择颜色）
4. 确认 → 扣减 50 墨水 → `POST /api/pins` → D1 写入
5. 新图钉立即显示在地图上

### 7.3 图钉渲染

图钉使用 MapLibre GL 的 `Marker` API 渲染，每个图钉包含：
- **气泡 tooltip**：默认显示前 10 字预览，hover 时展开完整消息 + 作者 + 时间
- **图钉图标**：CSS 旋转 45 度的圆角矩形，颜色可选
- 仅在 zoom >= 20 时加载和显示

### 7.4 图钉数据存储

```sql
CREATE TABLE map_pins (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES users(id),
  user_name TEXT NOT NULL DEFAULT 'Anonymous',
  lng REAL NOT NULL, lat REAL NOT NULL,
  message TEXT NOT NULL DEFAULT '',
  color TEXT NOT NULL DEFAULT '#E63946',
  created_at INTEGER NOT NULL DEFAULT (unixepoch())
);
```

## 八、认证系统

### 8.1 概述

使用 Lucia Auth v3 + D1Adapter，密码哈希使用 PBKDF2（Web Crypto API）。

**为什么不用 Argon2？** Edge Runtime / Cloudflare Workers 不支持 `@node-rs/argon2`（需要 WASM/native binding），因此统一使用 PBKDF2，600,000 次迭代，符合 OWASP 2024 推荐。

### 8.2 认证流程

```
注册: /register → Server Action →
  1. 验证输入 (邮箱/用户名/密码长度)
  2. 检查邮箱 + 用户名唯一性 (D1)
  3. PBKDF2 哈希密码 (600K 迭代)
  4. INSERT users (D1)
  5. lucia.createSession() → Set-Cookie → redirect /canvas

登录: /login → Server Action →
  1. D1 查询用户
  2. PBKDF2 验证密码 (时序安全比较)
  3. lucia.createSession() → Set-Cookie → redirect /canvas

画布访问: /canvas (无需登录可浏览)
  → 尝试绘画时弹出 AuthDialog
  → POST /api/drawings 时 validateSession() 验证

登出: POST /logout →
  lucia.invalidateSession() → 清除 Cookie → redirect /login
```

### 8.3 Guest 模式

画布页面 (`/canvas`) **不需要认证**即可访问。Guest 用户可以：
- 浏览地图
- 查看所有笔画和图钉
- 不能绘画（点击画笔时弹出登录弹窗）
- 不能放置图钉

## 九、实时同步系统

### 9.1 架构

```
SyncManager (Persistence)
  ├─ REST API (POST /api/drawings)    ← 主持久化通道
  ├─ TileManager (GET /api/drawings)  ← 瓦片化按需加载
  └─ OfflineQueue (IndexedDB)         ← 离线缓冲
```

### 9.2 同步流程

```
新笔画完成:
  1. POST /api/drawings → D1 持久化 (永远执行)
  2. DurableObjectClient.send() → WebSocket 广播给房间内其他人
  3. 若离线 → OfflineQueue.enqueue() → 重连后自动 flush

收到远程笔画:
  DurableObjectClient.onMessage() → engine.addExternalStroke()
  → StrokeManager.add() → RenderPipeline.requestRender()
```

### 9.3 房间划分

按 zoom-14 瓦片 key 分房间（如 `14/13358/6182`）。视口中心移动到新瓦片时自动切换房间。

### 9.4 瓦片化缓存策略 (Tile-based Caching)
用户移动地图时，计算视野内包含的 Tile 坐标，仅请求未加载或过期的 Tile 数据。

### 9.6 离线支持

```
在线 → WebSocket 广播 + REST 持久化
离线 → IndexedDB 队列 (idb-keyval)
重连 → drain 队列 → 按时间戳排序 → 批量发送
```

| 列 | 类型 | 说明 |
|----|------|------|
| id | TEXT PK | UUID v7 |

**索引：**
- `idx_drawings_bounds(min_lng, max_lng, min_lat, max_lat)` — 空间查询

**map_pins 表**

| 列 | 类型 | 说明 |
|----|------|------|
| id | TEXT PK | UUID v7 |
| user_id | TEXT FK → users | 作者 |
| user_name | TEXT | 作者名 |
| lng/lat | REAL | 经纬度 |
| message | TEXT | 留言 (<=50字) |
| color | TEXT | 图钉颜色 |
| created_at | INTEGER | Unix 秒 |

### 10.2 空间查询

D1 (SQLite) 没有 PostGIS，使用**包围盒粗过滤**：

```sql
SELECT * FROM drawings
WHERE min_lng <= ?maxLng AND max_lng >= ?minLng
  AND min_lat <= ?maxLat AND max_lat >= ?minLat
ORDER BY created_at DESC LIMIT 5000
```

精确的空间索引在客户端使用 `rbush` (R-tree) 完成。

