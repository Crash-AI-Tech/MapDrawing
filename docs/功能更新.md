# 功能更新与优化计划

> 最后更新：2025-07-10  
> 状态：**部分完成** — P0/P1/P2 大部分已实施，P3 待讨论

## 一、iOS / Web 现状差异总结

### 1. 绘制系统

| 维度 | iOS | Web |
|------|-----|-----|
| 渲染引擎 | Skia Canvas (SkPicture 缓存) | Canvas2D (OffscreenCanvas 位图缓存) |
| 画笔类型 | Pencil + Eraser（硬编码配置） | Pencil + Eraser（BrushRegistry 插件架构，对旧类型自动 fallback） |
| 颜色选择 | 54 色色板，不支持自定义颜色 | 54 色色板 + 自定义 Hex 输入 + 原生取色器 |
| 尺寸控制 | Stepper 按钮（±0.5） | Slider 滑块（±0.5） |
| 透明度范围 | 0.1 ~ 1.0（10 档） | 0.05 ~ 1.0（20 档） |
| 压感支持 | 无 | PressureAdapter（硬件压感 + 速度模拟 fallback） |
| 墨水显示 | 浮动水滴角标（绿→黄→红百分比） | 水平条形墨水条 |
| 工具栏位置 | 底部浮动（SafeAreaView） | 顶部居中固定 |
| 画笔面板 | 颜色网格 + Stepper 按钮（overlay） | Popover 弹出（颜色网格 + Slider + Hex 输入） |

### 2. 用户信息管理

| 维度 | iOS | Web |
|------|-----|-----|
| 认证框架 | Expo SecureStore + Bearer Token | Lucia Auth + HTTP-only Cookie |
| 登录方式 | 邮箱密码 + Apple Sign In + 游客模式 | 邮箱密码 + 游客模式 |
| 个人资料 | 全屏独立页面（头像上传、统计、设置） | Popover 面板（头像、统计、设置） |
| 统计项 | 图钉数、绘制数 | 图钉数、绘制数、浏览量 |
| 语言切换 | 不支持 | 支持（EN/ZH） |
| 认证流程 | 独立路由页面（login/register/forgot-password） | 多步骤弹窗（AuthDialog 内切换模式） |
| 头像上传 | expo-image-picker + FormData | 从 profile API 获取 |

### 3. 图钉（Pin）系统

| 维度 | iOS | Web |
|------|-----|-----|
| 图钉详情 | PinDetailModal（头像、用户名、时间、消息、举报/屏蔽） | **缺失** — 无图钉详情弹窗 |
| 地图提示 | MapPinTooltip（点击弹出悬浮卡片） | 无等效组件 |
| 放置面板 | KeyboardAvoidingView 底部面板 | 固定定位顶部面板 |
| 自定义颜色 | 不支持 | 支持 Hex 输入 |
| 举报/屏蔽 | Compliance 工具（reportContent + blockUser） | 有 Compliance 基础设施但缺少入口 UI |

### 4. 架构与性能

| 维度 | iOS | Web |
|------|-----|-----|
| 状态管理 | React hooks + useRef（strokes 不触发渲染） | Zustand stores（drawing/auth/pin/UI/sync） |
| 空间索引 | Tile-based (TileRenderer 地理网格) | R-tree (rbush) 空间查询 |
| 离线队列 | AsyncStorage 持久化 | IndexedDB (idb-keyval) 持久化 |
| 手势识别 | RN GestureHandler (Pan) | GestureRecognizer（多指手势、三指撤销） |
| LRU 驱逐 | Drawings 2500 / Pins 600 上限 | 类似策略 |

## 二、优化计划

### Phase 1：功能对齐（高优先级）

> 目标：消除两端功能差距，确保核心体验一致

#### 1.1 Web 端补齐图钉详情
- **任务**：在 Web 端实现 `PinDetailModal` 组件
- **内容**：点击图钉弹出详情卡片，展示头像、用户名、时间、消息内容
- **包含**：举报 (Report) 和屏蔽 (Block) 按钮，接入现有 Compliance 基础设施
- **参考**：iOS 端 `PinDetailModal.tsx` + `MapPinTooltip.tsx`

#### 1.2 iOS 端支持自定义颜色
- **任务**：在 iOS 颜色面板增加自定义 Hex 输入框
- **内容**：在 54 色网格下方添加一个 Hex 输入区域 + 颜色预览
- **对齐**：与 Web 端 ColorPicker 的自定义颜色功能保持一致

#### 1.3 透明度范围统一
- **任务**：统一两端透明度步进精度
- **方案**：将两端统一为 0.05 ~ 1.0（20 档），写入 `@niubi/shared`
- **影响**：iOS 需将 Stepper 步进从 0.1 改为 0.05

#### 1.4 iOS 端增加语言切换
- **任务**：在 iOS profile 页面增加语言选择（中/英）
- **参考**：Web 端 AuthDialog 中的 `LanguageToggle` 组件

#### 1.5 Web 端增加 Apple Sign In
- **任务**：Web 端认证流程增加 "Sign in with Apple" 按钮
- **前提**：需要 Apple Developer 配置 Web 域名的 Service ID
- **优先级**：中（取决于 Apple 开发者配置进度）

---

### Phase 2：界面优化（中优先级）

> 目标：提升视觉品质和交互一致性

#### 2.1 iOS 工具栏交互升级
- **任务**：将 Size / Opacity 的 Stepper 按钮改为 Slider
- **原因**：Slider 操作更直观、调节速度更快，与 Web 端保持一致
- **保留**：Stepper 精确微调模式（长按 Slider 两端可精确 ±0.05）

#### 2.2 iOS 颜色面板重设计
- **任务**：将当前 overlay 形式改为类似 Web 的 Popover/底部弹出面板
- **改进**：
  - 增加"最近使用"颜色行（最多 6 个）
  - 增加自定义颜色输入（Phase 1.2 合并实现）
  - 面板外点击关闭

#### 2.3 Web 墨水显示增强
- **任务**：在 InkBar 旁增加数值百分比显示
- **补充**：低墨水时增加脉动动画提醒（< 20%）
- **参考**：iOS 端水滴角标的百分比设计

#### 2.4 统一图标风格
- **任务**：审查两端使用的图标集，统一视觉风格
- **iOS**：当前使用 SF Symbols / Expo Icons
- **Web**：当前使用 Lucide Icons
- **方案**：确保相同功能使用视觉相似的图标，必要时自定义 SVG

#### 2.5 Web 端 PinPlacer 位置优化
- **任务**：将 PinPlacer 从顶部固定改为跟随地图中心点附近浮动
- **原因**：减少视线移动距离，提升放置体验

---

### Phase 3：新功能增加

> 目标：引入新功能提升产品吸引力
#### 3.1 绘画半透明功能
- **任务**：新增一个按钮，点击之后，当前显示的图案变成半透明，方便看到底下的地图细节
- **实现**：在 Stroke 数据结构中增加 `isTransparent` 属性，渲染时根据该属性调整 alpha 值（如 0.5）
- **双端**：iOS + Web 同步实现

#### 3.4 图钉互动功能
- **任务**：为图钉增加点赞 / 回复功能
- **后端**：新增 `pin_reactions` 和 `pin_replies` 表
- **前端**：在 PinDetailModal 中增加点赞按钮和简单回复列表
- **双端**：iOS + Web 同步实现

#### 3.5 绘制回放功能
- **任务**：支持回放当前视口内的绘制历史（按时间戳排序逐笔重绘）
- **入口**：工具栏增加"回放"按钮
- **实现**：利用现有 stroke 的 `createdAt` + `points` 数据，按时间序列动画重绘
- **双端**：优先 Web 实现，iOS 跟进

#### 3.6 用户画廊 / 作品集
- **任务**：在用户 Profile 中增加"我的作品"页签
- **内容**：展示用户历史绘制的缩略图列表（瓦片截图）
- **操作**：点击可在地图上定位到该绘制的位置（飞行动画）
- **后端**：API 增加 `GET /api/profile/drawings` 按用户 ID 分页查询

---

### Phase 4：用户体验提升

> 目标：打磨细节，优化流畅度和反馈感


#### 4.2 iOS 端手势增强
- **任务**：增加多指手势支持
- **新增**：
  - 双指缩放（当前已通过地图原生支持，绘制时也应支持）
  - 三指轻扫 → 撤销（对齐 Web 端的 GestureRecognizer）
- **冲突处理**：绘制中检测到多指时取消当前笔画

#### 4.3 离线体验优化
- **任务**：增加明确的网络状态 UI 指示
- **内容**：
  - 工具栏显示同步状态图标（已同步 / 同步中 / 离线排队）
  - 离线时仍可绘制 / 放图钉，上线后自动提交
  - iOS 和 Web 已有 OfflineQueue 基础，增加 UI 层反馈

#### 4.4 墨水系统 UX 改进
- **任务**：优化墨水不足时的反馈
- **改进**：
  - 墨水 < 20% 时画刷颜色渐变为灰色，提前预警
  - 墨水耗尽时画面轻微震动反馈（iOS haptic + Web CSS shake）
  - 增加墨水恢复倒计时展示（"还需 X 秒恢复 1 点墨水"）


#### 4.6 Web 端键盘快捷键完善
- **任务**：增加更多快捷键
- **新增**：
  - `B` — 切换画笔面板
  - `E` — 切换橡皮擦
  - `H` — 手动模式
  - `P` — 图钉模式
  - `[` / `]` — 减小 / 增大画笔尺寸
  - `Esc` — 退出当前模式 / 关闭面板
- **UI**：在工具栏 tooltip 中显示快捷键提示

## 四、技术依赖说明

- **`@niubi/shared` 包**：Phase 1 的常量统一工作需优先更新此包，两端同步更新引用
- **后端 API**：Phase 3.4（图钉互动）和 3.6（用户画廊）需要新增数据库表和 API 端点
- **Apple Developer**：Phase 1.5 需要预先完成 Web 域名的 Apple Service ID 配置
- **Drizzle 迁移**：新增表结构需要创建对应的 migration 文件
